# CLAUDE.md — bron_mod (модуль бронирования)

Отдельный, самостоятельный **модуль бронирования** для размещения на VPS (Ubuntu 24) и подключения к сайтам на Tilda. Базируется на готовом модуле из проекта-донора `azov-resort/`, но собирается как независимый продукт.

## ⚠️ Обязательные правила работы в этом проекте

1. **Читай `PROJECT.md` перед началом задачи.** Это карта проекта (архитектура, модели, маршруты, решения). Не нужно заново перелопачивать код. Если правка влияет на содержимое `PROJECT.md` — **сразу обнови его** (код главнее: при расхождении правь доку).

2. **Веди живой план в `LIVE_PLAN.md`.** Перед работой свериться с планом, по ходу — отмечать прогресс (статусы, галочки, заметки о решениях). План отражает текущее состояние, а не историю.

3. **Используй подходящие скилы (Skill).** Под стек проекта (Next.js + React + TypeScript + Prisma/PostgreSQL, фронтенд-вёрстка) релевантны: `fullstack-developer`, `fastapi-patterns` (если появится отдельный backend), `design-taste-frontend` / `frontend-design-direction` (вёрстка карточек/страниц), `test-driven-development` (перед реализацией фич), `code-review` / `simplify` (после правок). Перед реализацией фичи проверь, нет ли подходящего скила, и задействуй его.

4. **Не игнорируй спорные/неясные моменты ТЗ.** Если в задаче есть архитектурная развилка или непонятный пункт — обсуждай с владельцем (AskUserQuestion), не решай молча.
